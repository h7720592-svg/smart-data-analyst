## 一、项目背景与意义

### 1.1 选题背景

在数据驱动的时代，数据分析能力已成为各行各业的核心需求。然而，传统的数据分析工具（如 Excel、Tableau）存在操作门槛高、学习成本大的问题；编写 Python 代码进行数据分析又要求使用者具备编程能力。大语言模型（LLM）的快速发展为解决这一问题提供了新的可能——用户可以用自然语言描述分析需求，由 AI 自动生成分析代码并执行。

### 1.2 项目意义

本项目将 **LLM 的自然语言理解能力** 与 **传统数据分析工具（Pandas、Plotly）** 相结合，构建一个零门槛的智能数据分析平台。用户只需上传数据文件，即可通过自然语言对话完成数据探索、统计分析和可视化图表的生成。项目完全开源，基于 MIT 协议发布，任何人都可以自由使用、修改和分发。

### 1.3 课程关联

本项目涵盖开源技术课程的核心知识点：
- **开源协作**：完整的 GitHub 项目结构（README、CONTRIBUTING、CI/CD、Issue/PR 模板）
- **软件工程**：模块化架构设计、单元测试、安全沙箱、错误处理
- **AI 技术**：LLM API 调用、Prompt Engineering、代码生成与安全执行
- **数据可视化**：Plotly 交互式图表、数据画像、报告导出

## 四、核心模块实现

### 4.1 数据加载模块 (`src/data_loader.py`)

**功能**：支持 CSV、Excel (.xlsx/.xls)、JSON 三种格式的文件读取。

**关键技术**：
- **编码自动检测**：使用 `chardet` 库进行初始检测，置信度低于 0.7 时依次尝试 UTF-8 → GBK → GB2312 → GB18030 → latin-1 编码链，解决中文 CSV 文件乱码问题
- **分隔符自动识别**：统计首行中逗号、制表符、分号、竖线的出现次数，自动选择最可能的 delimiter
- **列名规范化**：自动去除列名首尾空格
- **Schema 摘要生成**：为 LLM 构建数据上下文（列名、类型、缺失率、样本值）


### 4.2 数据画像模块 (`src/profiler.py`)

**功能**：自动数据探索分析和质量问题检测。

**关键技术**：
- **统计摘要**：每列的均值、中位数、标准差、四分位数、偏度等
- **问题检测**：
  - 缺失值：超过 30% 标记为问题
  - 异常值：IQR 方法（Q1 - 1.5×IQR, Q3 + 1.5×IQR）
  - 偏度：|skew| > 1 提示偏态分布
  - 常数列：唯一值数量 ≤ 1
  - 重复行检测
- **大数据优化**：超过 20 万行自动采样到 10 万行进行分析

### 4.3 LLM 客户端模块 (`src/llm/client.py`)

**功能**：统一的多供应商 LLM 调用接口。

**关键技术**：
- **统一接口**：DeepSeek、OpenAI、Groq、Ollama 均为 OpenAI 兼容 API，仅需切换 `base_url`
- **思考模式**：DeepSeek V4 Flash 支持通过 `extra_body={"thinking": {"type": "enabled"}}` 开启深度推理
- **JSON 解析**：支持直接 JSON、Markdown 代码块、文本内嵌 JSON 三种响应格式的自动提取

### 4.4 代码执行沙箱 (`src/llm/code_executor.py`) 🔴

**功能**：安全执行 LLM 生成的 Python 代码。

**三层安全防御**：

| 层级 | 机制 | 防护内容 |
|------|------|----------|
| 1. AST 白名单 | 解析代码 AST，检查所有 `import` 语句 | 阻止 `os`、`subprocess`、`requests`、`socket` 等危险模块 |
| 2. Restricted Builtins | 移除 `open()`、`exec()`、`eval()`、`compile()`、`__import__()` | 阻止文件 I/O 和动态代码执行 |
| 3. 进程超时 | `multiprocessing.Process` + 30 秒 `join(timeout)` | 防止死循环和资源耗尽 |

**错误恢复机制**：代码执行失败后，将错误信息反馈给 LLM 进行自动修正，最多重试 2 次。

### 4.5 Prompt 工程设计 (`src/llm/prompts.py`) 🔴

**功能**：构建发送给 LLM 的完整对话上下文。

**设计要点**：
- **System Prompt 结构**：角色定义 → 数据 Schema 注入 → 响应格式规定 → 代码生成规则 → 安全规则 → Few-shot 示例
- **Few-shot 示例**：覆盖柱状图、异常值检测、分布图、统计查询、相关性分析 5 种场景
- **中文优化**：System Prompt 使用英文（LLM 理解更好），Assistant 回复使用中文
- **NLP 引导**：明确告知 LLM 可使用 `jieba`、`Counter` 进行基础情感分析和主题建模

### 4.6 图表系统 (`src/viz/`)

**功能**：图表类型注册与渲染。

**支持的 14 种图表**：柱状图、横向柱状图、折线图、散点图、饼图、直方图、箱线图、热力图、面积图、小提琴图、密度热力图、旭日图、矩形树图、漏斗图

**统一主题**：所有图表应用一致的配色方案和字体设置。

### 4.7 报告导出 (`src/export/`)

**功能**：将分析结果导出为 HTML 和 PDF 报告。

**HTML 报告**：Jinja2 模板渲染，嵌入 Plotly.js CDN 实现交互式图表。
**PDF 报告**：WeasyPrint 转换，配置 CJK 字体回退链（微软雅黑 → 苹方 → Noto Sans CJK → 思源黑体）。

## 五、安全设计

LLM 生成的代码不可信，本项目实现了多层安全防护：

1. **AST 白名单验证**：在代码编译前遍历 AST 树，检查所有导入语句是否在白名单内。发现 `import os`、`from requests import get` 等危险导入直接拒绝执行，返回安全错误。

2. **Restricted Builtins**：构建受限的 `__builtins__` 字典，移除 `open()`、`exec()`、`eval()`、`compile()`、`__import__()` 等危险函数。`import` 语句内建的 `__import__` 调用不受影响（通过 AST 层单独拦截显式调用）。

3. **进程隔离与超时**：使用 `multiprocessing.Process` 在子进程中执行用户代码，主进程设置 30 秒 `join(timeout)`，超时则 `terminate()` + `kill()`。

4. **安全声明**：README 中明确标注此为课程项目，生产环境建议 Docker 容器化运行。

## 八、项目总结

### 8.1 成果

本项目成功构建了一个基于大语言模型的智能数据分析助手，实现了以下目标：

1. **零门槛数据分析**：用户无需编程知识，通过自然语言即可完成数据探索和可视化
2. **安全可靠**：三层代码沙箱确保 LLM 生成代码的安全执行
3. **多供应商支持**：兼容 DeepSeek、OpenAI、Groq、Ollama 等多种 LLM
4. **完整的开源体系**：规范的 GitHub 项目结构、CI/CD、文档、测试

### 8.2 技术亮点

- **Prompt Engineering**：精心设计的 System Prompt 和 Few-shot 示例，覆盖 14 种图表类型
- **自动纠错**：LLM 生成代码出错后自动将错误信息反馈给 LLM 进行修正
- **中文编码兼容**：完整的 GBK/GB2312/GB18030 编码检测链
- **NLP 能力**：集成 jieba 分词，支持基础情感分析和主题建模

### 8.3 不足与展望

1. **NLP 能力有限**：受沙箱限制，无法使用 sklearn、transformers 等高级 NLP 库，情感分析和主题建模依赖 LLM 自行实现的基础方法
2. **单用户模式**：Streamlit 的 session state 机制不支持多用户并发
3. **PDF 字体依赖**：WeasyPrint 的 CJK 字体需要系统预装
4. **未来改进方向**：Docker 容器化部署、数据库直连、流式输出、多轮对话记忆优化、支持更多数据源（MySQL、API）

- Python AST 代码分析与安全沙箱构建
